# take-waza アーキテクチャ

## 設計概要

クライアントサイド完結型のWebアプリケーション。
Rust/WASMで竹ひごの編みパターンをグラフ構造として計算し、Three.jsで3D描画する。

## コアモデル

- **ノード**: 竹ひごの交差点。位置座標(x,y,z)と交差タイプ（上下）を持つ
- **エッジ**: 竹ひご。ノード間を接続。幅・厚さ・素材パラメータを持つ
- **パターン**: ノードとエッジの配置規則。繰り返し単位（ユニットセル）で定義

## データフロー

```
[パターン選択 + パラメータ入力]
  → [Rust/WASM: グラフ生成・パラメトリック変形]
  → [Three.js: メッシュ生成・レンダリング]
  → [ステップ分解: 編み手順の順序付け]
```

## 主要な設計判断

- ADR記録予定: MVP段階ではRust/WASMの計算エンジンを独立crateとして分離
- ADR記録予定: パターンデータは静的JSON（MVP後にAPI化）

## 外部サービス連携

MVP段階では外部サービス連携なし（クライアントサイド完結）。
Phase 1でパターン共有API（Go + PostgreSQL + GCP Cloud Run）を追加予定。

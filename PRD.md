# PRD: take-waza

**作成者**: Ryusei | **日付**: 2026-03-30 | **ステータス**: ドラフト
**アイデアID**: #471 | **ドメイン**: 竹細工/竹工芸 | **難易度**: Medium

## 問題定義

竹細工の編み方パターンは師弟間の口伝や書籍の静止画に依存しており、初学者が完成形をイメージしながら設計することが困難。竹ひごの幅・厚さの変更が仕上がりにどう影響するかを事前にシミュレーションする手段がなく、試行錯誤に多大な材料と時間を要する。take-wazaは編みパターンをグラフ理論でモデル化し、パラメトリックな3Dプレビューで完成形を確認しながら設計できるCADライクなツールを提供する。

## 目標と成功指標

| 目標 | 指標 | ターゲット |
|------|------|----------|
| 伝統パターンのデジタル化 | 内蔵パターン数 | 10種以上（六つ目編み、網代編み、ござ目編み等） |
| インタラクティブ設計体験 | パラメータ変更から3Dプレビュー反映までの遅延 | 500ms以下 |
| 初学者の学習支援 | ステップバイステップ図解の生成成功率 | 95%以上 |
| ユーザー獲得 | MVP公開後1ヶ月のユニークユーザー | 50人 |
| 技術的差別化 | グラフ理論ベースの編みパターンモデル化の実装 | 完了 |

## スコープ外

- ユーザー認証・アカウント管理（MVPではローカル保存のみ）
- パターン共有API（Go API層はMVP後）
- 強度・柔軟性の物理シミュレーション（MVPでは3Dプレビューのみ）
- モバイルアプリ対応
- 多言語対応（MVPは日本語のみ）
- ECサイト連携・材料購入機能

## ユーザーストーリー

- 竹細工初学者として、伝統的な編み方パターンを3Dで確認したい。なぜなら書籍の2D図だけでは完成形をイメージできないから。
- 竹工芸愛好家として、竹ひごの幅・厚さを変えたときの仕上がりをプレビューしたい。なぜなら実際に編む前にパラメータを最適化して材料の無駄を減らしたいから。
- 竹細工教室の講師として、編み手順のステップバイステップ図解を生成したい。なぜなら生徒への指導資料を効率的に作成したいから。
- 竹工芸デザイナーとして、伝統パターンをベースにパラメータを変形させたオリジナルデザインを探索したい。なぜなら計算的にバリエーションを発見できれば創作の幅が広がるから。

## 技術要件

### 技術スタック

- **フロントエンド**: TypeScript / React + Three.js（3Dプレビュー）
- **計算エンジン**: Rust / WASM（編み構造のグラフ計算・シミュレーション）
- **API層**（MVP後）: Go
- **データベース**（MVP後）: PostgreSQL
- **インフラ**（MVP後）: GCP Cloud Run

### アーキテクチャ

MVPはフロントエンド + Rust/WASMのクライアントサイド完結型。Rustで編みパターンのグラフ計算を行い、WASMにコンパイルしてブラウザ上で実行。Three.jsで3Dレンダリング。

```
[React UI] <-> [Three.js 3D Renderer]
    |
[Rust/WASM 計算エンジン]
    |-- グラフ構造モデル（ノード: 交差点、エッジ: 竹ひご）
    |-- パラメトリック変形
    |-- ステップバイステップ分解
```

### 非機能要件

- パフォーマンス: パラメータ変更から3D更新まで500ms以下（10パターン、100ノード規模）
- セキュリティ: クライアントサイド完結（MVPではサーバー通信なし）
- スケーラビリティ: WASMバイナリサイズ 1MB以下

## 技術リスク

| リスク | 発生可能性 | 影響 | 軽減策 |
|--------|----------|------|--------|
| 竹ひごの交差パターンのグラフ表現が複雑化 | 中 | 高 | MVP段階では基本的な2D格子パターンから開始し、段階的に3D湾曲に対応 |
| Rust/WASM のビルド・デバッグ環境の複雑さ | 中 | 中 | wasm-pack を使い、計算エンジンを独立モジュールとして分離 |

| 伝統パターンの正確なモデル化に竹工芸の専門知識が必要 | 高 | 中 | 公開文献・書籍から基本10パターンを調査、正確性はコミュニティフィードバックで改善 |

## マイルストーン

### Phase 0: MVP（2週間）

1. **基盤セットアップ（2日）**: CI/CD、リンター、Rust/WASM ビルドパイプライン
2. **グラフデータモデル（3日）**: 竹ひご交差パターンのノードグラフ表現、基本パターン3種（六つ目、網代、ござ目）
3. **3Dプレビュー（4日）**: Three.js でのグラフ→3Dメッシュ変換、カメラ操作
4. **パラメータUI（2日）**: 竹ひご幅・厚さ・本数のスライダー、リアルタイムプレビュー更新
5. **ステップ図解（3日）**: 編み手順の分解アルゴリズム、ステップバイステップ表示

### Phase 1: 拡張（MVP後）

- パターンライブラリ拡充（10種 → 30種）
- パターン共有API（Go + PostgreSQL）
- エクスポート機能（SVG、PDF）
- 強度シミュレーション
